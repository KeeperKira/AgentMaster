# UI Спецификация — AgentMaster

> **Дата создания:** 13 апреля 2026

---

## 1. Технологический стек

| Компонент | Решение |
|---|---|
| **GUI-фреймворк** | Dear ImGui (immediate mode) |
| **Библиотека нод** | ImNodes (готовые ноды с портами и связями) |
| **Интеграция** | Win32 + ImGui (ImGui поверх Win32 окна) |
| **Язык** | C++20 |

### Почему ImGui + ImNodes?
- Готовый drag-and-drop из коробки
- Кастомная отрисовка через `ImDrawList`
- Zoom/Pan через масштабирование координат
- Минимум boilerplate-кода по сравнению с чистым Win32 GDI

## 2. Структура интерфейса

### 2.1. Двухпанельная компоновка

```
┌──────────────────────────────────────────────────────┐
│  Menu Bar: File (Exit) | Help (About)               │
├──────────┬───────────────────────────────────────────┤
│ Catalog  │           Canvas (ImNodes)                │
│          │                                           │
│ [Input]  │    [Input]──────[Router]────[Output]     │
│ [Text]   │       │                │                  │
│ [Triplet]│    [Triplet]───────────┘                  │
│ [Router] │                                           │
│ [Output] │                                           │
│          │                                           │
└──────────┴───────────────────────────────────────────┘
```

### 2.2. Каталог инструментов (левая панель)

- **Вид:** Вертикальный список с иконками/названиями
- **Поведение:** Drag из каталога → Drop на canvas = создание узла
- **Инструменты:**

| Инструмент | Входы | Выходы | Поля |
|---|---|---|---|
| **Input (Ввод)** | 0 | 1 | — (всегда на canvas) |
| **Output (Вывод)** | 1 | 1 | — (всегда на canvas) |
| **Text (Текст)** | 0 | 1 | Name, Text |
| **Triplet (Триплет)** | 3 | 1 | Name |
| **Router (Роутер)** | 1 | 1 | Name, URL, API_KEY |

## 3. Визуальный стиль узлов

### 3.1. Порты

- **Форма:** Кружки (circles)
- **Расположение:** Слева — входы, Справа — выходы
- **Правила соединений:**
  - Один выход → много входов
  - Один вход ← только один выход

### 3.2. Связи (линии между портами)

- **Тип:** Прямые линии (не кривые Безье)
- **Цвет:** Настраиваемый (по умолчанию — серый/белый)

---

## 4. Навигация по Canvas

| Действие | Управление |
|---|---|
| **Zoom** | Колёсико мыши |
| **Pan** | Средняя кнопка мыши (колёсико нажатие) + движение |

---

## 5. Управление узлами

### 5.1. Удаление

- **Кнопка:** Красный крестик (×) на узле
- **Альтернатива:** Выделение узла + клавиша Delete

### 5.2. НЕ реализуется на первом этапе

- [ ] Undo/Redo (Ctrl+Z / Ctrl+Y)
- [ ] Контекстное меню (ПКМ)
- [ ] Миникарта
- [ ] Дублирование узлов

---

## 6. Экспорт

- **Формат:** JSON
- **Содержимое:**
  1. Список узлов на canvas (тип, позиция, поля)
  2. Соединения между портами
  3. Значения всех полей (text, name, url, api_key)

> Подробности: см. [`ARCH_SPEC.md`](./ARCH_SPEC.md) — формат JSON, структуры данных.
