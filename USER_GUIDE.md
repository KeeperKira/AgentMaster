# Руководство пользователя — AgentMaster

> **Версия:** 14 апреля 2026
> **Тип приложения:** Визуальный редактор схем AI-агентов

---

## Что такое AgentMaster?

AgentMaster — программа для создания схем AI-агентов в виде блок-схемы (node-based editor). 
Вы расставляете блоки (узлы) на рабочем поле и соединяете их линиями, определяя поток данных от входа к выходу.

---

## Запуск

```
x64\Debug\AgentMaster.exe
```

При запуске на рабочем поле автоматически появляются два фиксированных узла:
- **Input** (Ввод) — точка входа запроса
- **Output** (Вывод) — точка выхода ответа

### Левая панель — Каталог инструментов

Список доступных типов узлов. Клик по названию → клик на рабочем поле = создание узла.

### Правая панель — Canvas (рабочее поле)

Здесь располагаются узлы и связи. Можно перемещать узлы, создавать и удалять связи.

---

## Создание узлов

1. Кликните по названию нужного инструмента в **каталоге** (левая панель)
2. Кликните по **рабочему полю** (правая панель) в нужное место
3. Узел появится в точке клика

> Для отмены режима размещения нажмите **Escape** или **ПКМ** (правая кнопка мыши).

---

## Типы узлов

| Узел | Входы | Выходы | Поля | Описание |
|---|---|---|---|---|
| **Input** | 0 | 1 | Имя | Точка входа. Всегда на canvas. Нельзя удалить |
| **Output** | 1 | 1 | Имя | Точка выхода. Всегда на canvas. Нельзя удалить |
| **Text** | 0 | 1 | Имя, Text | Текстовый блок. Поле Text — многострочное поле для ввода текста |
| **Triplet** | 3 | 1 | Имя | Блок контекста. Три входа для субъекта, предиката, объекта |
| **Router** | 1 | 1 | Имя, Model, URL, API Key | Роутер к внешнему API. Model — название модели (qwen/qwen3.6plus), URL — адрес эндпоинта, API Key — ключ авторизации |
| **Concat** | 2 | 1 | Имя, Wait | Объединение двух входов в один выход. Wait — флаг ожидания |

### Редактирование имени

Имя узла редактируется прямо в **строке заголовка** каждого узла — текстовое поле рядом с названием типа.

---

## Связи между узлами

### Создание связи

1. Зажмите левую кнопку мыши на **выходном порту** (правый кружок, «Out»)
2. Потяните линию к **входному порту** (левый кружок, «In») другого узла
3. Отпустите кнопку мыши

> Направление: **только от выхода ко входу**. Наоборот — нельзя.

### Удаление связи

Два способа:

- **ПКМ на линии** — наведите на связь (линию) и нажмите правую кнопку мыши
- **ПКМ на порту** — наведите на кружок порта и нажмите правую кнопку мыши. Удалятся **все связи** к этому порту

---

## Удаление узлов

- Нажмите на красную кнопку **«X»** в заголовке узла
- Узлы **Input** и **Output** удалить нельзя (они фиксированные)

---

## Перемещение узлов

- Зажмите левую кнопку мыши на узле и перетащите

---

## Навигация по рабочему полю

| Действие | Управление |
|---|---|
| **Zoom** | Колёсико мыши |
| **Pan** (сдвиг) | Средняя кнопка мыши (нажатие на колёсико) + движение |

---

## Сохранение и загрузка

### Сохранить схему

1. **File → Save JSON...**
2. Выберите файл для сохранения (`.json`)
3. Схема сохранится: узлы, позиции, поля и связи

### Загрузить схему

1. **File → Load JSON...**
2. Выберите ранее сохранённый `.json` файл
3. Canvas очистится и загрузит схему из файла

---

## Горячие клавиши

| Клавиша | Действие |
|---|---|
| **Escape** | Отмена режима размещения узла |

## Формат JSON

```json
{
  "nodes": [
    {
      "id": 1,
      "type": "input",
      "x": 100,
      "y": 200,
      "fields": { "name": "" }
    },
    {
      "id": 2,
      "type": "router",
      "x": 400,
      "y": 200,
      "fields": {
        "name": "Мой роутер",
        "url": "https://api.example.com",
        "api_key": "sk-xxx"
      }
    }
  ],
  "connections": [
    {
      "from_node": 1,
      "from_port": 0,
      "to_node": 2,
      "to_port": 0
    }
  ]
}
```
